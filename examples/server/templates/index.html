Justin's Final Year Project

As part of my final year project, I'm experimenting with a new technique for designing hacking challenges for educational use.

I'd really appreciate your help! I've designed a simple reverse-engineering challenge, which you can see below, in two parts - by solving each part, you'll get half of a flag (in the form FLAG{...}). The binaries are Linux-only, but you don't need Linux to solve them if you don't want!

If you decide to do this, please don't discuss the challenges with anyone else until you've both submitted the survey - that'll help me get the most accurate results possible!

Tools

To solve these challenges you can use any tools you like! But, if you're sure where to get started, try out some of these suggestions!

  • Ghidra, a reverse-engineering toolset (with all the bells and whistles)
  • objdump and readelf, standard Linux tools for disassembling and inspecting programs
  • strace and ltrace, to inspect system and library calls of running programs
  • GDB, the standard Linux debugger

You definitely don't need to use all/any of these tools, but they can be a nice place to start!
